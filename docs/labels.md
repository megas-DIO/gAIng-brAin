# Issue Label Taxonomy

## Priority
- `priority:critical` – immediate attention required.
- `priority:high` – high impact, scheduled soon.
- `priority:medium` – default priority.
- `priority:low` – nice-to-have.

## Type
- `type:feature` – new capability or behavior.
- `type:bug` – defect or regression.
- `type:docs` – documentation changes.
- `type:chore` – maintenance or tooling.
- `type:ci` – CI/CD pipeline work.

## Area
- `area:api` – backend services.
- `area:frontend` – dashboard UI.
- `area:mcp` – Model Context Protocol.
- `area:database` – Supabase/Postgres schema.
- `area:infra` – deployments, workflows.

## Status
- `status:needs-triage`
- `status:in-progress`
- `status:blocked`
- `status:ready-for-review`

## Guidelines

- Apply **one** `type:*` label to every issue.
- Apply **one** `priority:*` label based on impact.
- Add at least one `area:*` label.
- Use `status:*` to reflect current execution state.
